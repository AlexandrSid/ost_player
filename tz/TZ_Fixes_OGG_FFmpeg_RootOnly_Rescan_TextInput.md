# ТЗ: исправления OGG (ffmpeg fallback) + UX для папок/ввода + rescan-before-play

## 0) Контекст

В текущей сборке:

- **Все `.ogg` не воспроизводятся** (в TUI отображается `failed to decode ...: end of stream`), при этом **`.mp3` воспроизводятся**.
- При удалении папок из списка, их треки **могут продолжать воспроизводиться**, т.к. очередь строится из **кэшированного индекса** без обязательного перескана.
- В диалоге ввода (например “Добавить папку”) есть поле “как для ввода”, но **введённый текст не отображается** и **не видно курсора**.
- Нужно добавить пер‑папочную опцию: **“только файлы в корне папки”**.

Это ТЗ предназначено для агента‑оркестратора: фикс аудио + 2 UX‑изменения + изменение поведения Play.

## 1) Цели

- **OGG**: обеспечить воспроизведение `.ogg` через **fallback на `ffmpeg`** (т.к. текущий декодер не справляется со всеми `.ogg`).
- **Папки**: для каждой папки в списке хранить флаг **root-only** (“только корень”), по умолчанию **включён**.
- **Play**: при нажатии “воспроизвести” сначала делать **перескан активных каталогов**, затем строить очередь по свежему индексу.
- **TextInput**: сделать **видимый ввод** (показывать введённый текст) и **видимый курсор**.

## 2) Область изменений (ориентиры по текущему коду)

Ожидаемые точки интеграции (актуально на момент анализа):

- Декодирование: `app/src/audio/mod.rs` (`decode_file()` использует `rodio::Decoder::new(reader)`).
- Загрузка очереди: `app/src/tui/app.rs` (обработка `Action::PlayerLoadFromLibrary` → `PlayerCommand::LoadQueue { tracks, start_index }`).
- Список папок в конфиге: `app/src/config/mod.rs` (`AppConfig.folders: Vec<String>`).
- Сканирование: `app/src/indexer/scan.rs` (`scan_library(...)` и обход каталогов).
- UI главного меню: `app/src/tui/ui.rs` (рендер списка папок) и `app/src/tui/screens/main_menu.rs` (клавиши add/remove/play/rescan).
- Виджет ввода: `crate::tui::widgets::TextInput` (используется из `main_menu.rs` при добавлении папки).

## 3) Требование A: OGG через ffmpeg fallback

### 3.1 Поведение

- При попытке воспроизведения трека:
  - если расширение **не `.ogg`** → использовать текущий путь декодирования (Rodio/Symphonia).
  - если расширение **`.ogg`**:
    - сначала можно попробовать текущий декодер (опционально),
    - при ошибке декодирования **обязательно** пробовать **ffmpeg fallback**.
- Если ffmpeg fallback тоже не сработал:
  - пропустить трек, перейти к следующему,
  - показать ошибку в `Last error` (включая, что пытались `ffmpeg` и почему не вышло).

### 3.2 Реализация ffmpeg fallback (общая)

- `ffmpeg` должен декодировать аудио в поток PCM, который читается приложением:
  - допустимо: `wav` в stdout (`-f wav pipe:1`) или raw PCM (`-f s16le`/`f32le`).
- Для portable‑сценария допускается:
  - **опциональная фича** Cargo (например `ffmpeg`) и/или
  - поиск `ffmpeg.exe` рядом с бинарником (в папке приложения) и/или в `PATH`.
- Ошибки:
  - если `ffmpeg` не найден → сообщение “ffmpeg not found (fallback required for ogg)” + путь, где искали.
  - если `ffmpeg` завершился с ошибкой → отобразить stderr (обрезать до разумного размера).

### 3.3 Критерии готовности (OGG)

- На Windows 11:
  - `.ogg` файлы воспроизводятся (минимум: несколько разных `.ogg` из разных источников).
  - Если `ffmpeg` отсутствует, пользователь видит понятную ошибку и приложение не падает.
  - Ошибка декодирования не “ломает” очередь: приложение продолжает играть следующий трек.

## 4) Требование B: пер‑папочная опция “только файлы в корне папки”

### 4.1 Модель данных и дефолты

- Опция применяется **к каждой папке отдельно**.
- По умолчанию для новых папок: `root_only = true`.
- Нужно поддержать миграцию со старого конфига:
  - если в YAML папки представлены как список строк → считать `root_only = true`.

Предложенная структура (эквивалент допускается):

- `FolderEntry`:
  - `path: String`
  - `root_only: bool`
- `AppConfig.folders: Vec<FolderEntry>`

### 4.2 Отображение в UI списка папок

- Статус должен отображаться **между номером** и **путём**:
  - `↓` если `root_only = true`
  - `○` если `root_only = false` (значит рекурсивное сканирование)

Пример строки:

- ` 1. ↓ D:\Music\GameOST`
- ` 2. ○ D:\Music\BigCollection`

Допускается предложить более читаемую пару символов, но нужно сохранить идею “между номером и путём”.

### 4.3 Управление флагом из UI

- Добавить действие “toggle root-only” для выбранной папки:
  - предложенная горячая клавиша: `t` (если не конфликтует).
- Изменение должно:
  - сохраняться в `config.yaml` автоматически,
  - обновлять отображение списка.

### 4.4 Влияние на сканирование

- Если `root_only = true`, сканер должен искать файлы **только в корне этой папки**, без обхода подпапок.
- Если `root_only = false`, текущий рекурсивный обход сохраняется.

### 4.5 Критерии готовности (root-only)

- Для папки с подпапками:
  - при `↓` в индекс попадают только файлы верхнего уровня,
  - при `○` в индекс попадают файлы из подпапок.

## 5) Требование C: rescan-before-play (устранить “старую очередь”)

### 5.1 Проблема

Сейчас воспроизведение строится из уже загруженного/кэшированного индекса. Поэтому:

- если папка была удалена из списка, но индекс не пересканен, её треки всё ещё присутствуют в очереди.

### 5.2 Новое поведение

- При нажатии “воспроизвести” (как минимум в главном меню, а также в других местах где инициируется `LoadFromLibrary`):
  1) запустить перескан **активных** папок,
  2) дождаться завершения,
  3) построить очередь из свежего индекса,
  4) отправить в плеер `LoadQueue`.

Если скан уже идёт:

- не запускать второй параллельный скан,
- запомнить “play pending” и выполнить пункты 3–4 после завершения текущего скана.

### 5.3 Критерии готовности (rescan-before-play)

- Удаляем папку из списка, нажимаем Play:
  - треки из удалённой папки **не** попадают в новую очередь.
- Play работает предсказуемо: пользователь видит статус “Scanning…” и затем старт воспроизведения.

## 6) Требование D: TextInput — показывать ввод и курсор

### 6.1 Проблема

Сейчас при открытии окна ввода (например “Добавить папку”):

- пользователь может вводить/вставлять, но
- **на экране не отображается текущий буфер** и **не видно курсор**.

### 6.2 Новое поведение

- Виджет ввода должен:
  - отображать текущий текст буфера,
  - отображать курсор (например блок/подчёркивание) в позиции ввода,
  - корректно обрабатывать:
    - ввод символов,
    - Backspace,
    - стрелки Left/Right (если поддерживаются),
    - Enter (Submit),
    - Esc (Cancel).

### 6.3 Критерии готовности (TextInput)

- При вводе с клавиатуры текст виден.
- При вставке из буфера текст виден.
- Курсор виден всегда (в том числе когда строка пустая).

## 7) Нефункциональные требования и ограничения

- Не ломать portable‑поведение.
- Не вводить “тяжёлые” обязательные зависимости. `ffmpeg` допускается как внешний бинарник/опциональная фича.
- Все ошибки должны попадать в статус/`Last error`, без паник и падений.

## 8) Мини‑тест план

- **OGG**:
  - воспроизвести 3–5 разных `.ogg` (включая проблемные из игровых папок),
  - отключить/удалить `ffmpeg` → убедиться в понятной ошибке.
- **root-only**:
  - папка с файлами в корне и в подпапке → проверить `↓` vs `○`.
- **rescan-before-play**:
  - добавить папку → скан → play,
  - удалить папку → play → убедиться, что треков из неё нет.
- **TextInput**:
  - добавить папку вручную (ввод),
  - добавить папку через paste.
