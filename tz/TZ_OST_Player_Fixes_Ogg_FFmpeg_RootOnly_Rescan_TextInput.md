# Техническое задание (ТЗ): исправления декодирования OGG + улучшения UX TUI

## 0) Контекст
В текущей версии:
- `.mp3` воспроизводятся, но **все `.ogg` падают** с ошибкой декодирования (в TUI показывается `failed to decode ...: end of stream`).
- При удалении каталогов из списка, треки из них могут продолжать воспроизводиться, т.к. очередь строится по устаревшему кэшу индекса.
- В диалоге ввода (например “Добавить папку”) пользовательский ввод **не отображается** (поле “ждёт”, но текст и курсор не видны).
- Нужна пер‑папочная опция: **“только файлы в корне папки”** (не заходить в подпапки) с отображением статуса в списке папок.

Цель: внести исправления так, чтобы приложение снова было пригодно для OST из игровых папок (Steam и т.п.), сохранив portable‑подход.

## 1) Область работ (Scope)
Нужно реализовать и проверить:
1) **OGG decode fix**: fallback через `ffmpeg` для `.ogg` (и/или при неудачном decode) с корректной подачей PCM в существующий аудио‑пайплайн.
2) **Опция на папку**: “только файлы в корне папки” (по умолчанию включена) + отображение индикатора в списке папок + возможность переключать.
3) **Rescan-before-play**: при нажатии “воспроизвести” сначала перескан активных каталогов, потом построение очереди/старт.
4) **TextInput UX**: видимый буфер ввода + курсор (caret) в окне ввода значений.

## 2) Требование 1: исправить поддержку `.ogg` через fallback `ffmpeg`

### 2.1 Функциональные требования
- При попытке воспроизведения `.ogg` приложение должно **успешно воспроизводить** типичные игровые `.ogg` (в т.ч. Opus/Vorbis внутри контейнера OGG).
- Если текущий встроенный decode (через `rodio::Decoder`) не справился, должен сработать fallback через `ffmpeg`.
- При отсутствии `ffmpeg` (или невозможности запустить процесс) приложение должно:
  - показать понятный `Last error`/status (что нужен `ffmpeg` для `.ogg`),
  - не падать целиком,
  - продолжить работу с поддерживаемыми форматами (например `.mp3`).

### 2.2 Нефункциональные требования
- Fallback через `ffmpeg` должен быть:
  - **опциональным** (рекомендуется оформить через Cargo feature, чтобы можно было собирать “без ffmpeg”),
  - устойчивым к длинным путям Windows (включая `\\?\`),
  - без заметных задержек UI (процесс запускается в playback потоке/бэкенде, UI получает события как сейчас).
- Устранить/минимизировать “небольшую хрипоту” при воспроизведении:
  - Если хрипота вызвана некорректным форматом PCM/ресэмплингом, обеспечить корректную конвертацию и стабильный поток семплов для Rodio sink.

### 2.3 Предлагаемое поведение (нормативное)
- Алгоритм decode:
  - Для не-OGG: как сейчас (`rodio::Decoder`).
  - Для `.ogg`:
    - попытка decode штатно (опционально),
    - если ошибка — fallback через `ffmpeg`, декод в PCM и подача в sink.
- Формат PCM для пайпа (предпочтение):
  - `s16le` или `f32le` (выбрать тот, который проще/надёжнее для интеграции с rodio Source),
  - явные параметры: sample rate + channels (получать из ffmpeg/потока).

### 2.4 Диагностика ошибок
- Сообщение об ошибке (в `Last error`) должно различать:
  - “штатный декодер не справился”,
  - “fallback ffmpeg не доступен / не запускается / завершился с ошибкой”.

## 3) Требование 2: опция “только файлы в корне папки” для каждой папки

### 3.1 Модель данных и дефолт
- Опция применяется **к каждой папке в списке отдельно**.
- Дефолт: **включено** для всех существующих и новых папок.
- Значение должно сохраняться в `./data/config.yaml` вместе с путём папки.

### 3.2 Поведение сканирования
- Если опция включена: индексатор **не заходит в подпапки** выбранной папки.
- Если опция выключена: индексатор работает рекурсивно (как сейчас).

### 3.3 UI отображение и переключение
- В главном меню (список папок) отображать символ **между номером и путём**:
  - `↓` — “только корень” (включено),
  - `○` — “рекурсивно” (выключено).
- Добавить действие/хоткей для переключения флага у выделенной папки (например `t`), с автосохранением конфига.

## 4) Требование 3: rescan-before-play (чтобы не играли удалённые папки)

### 4.1 Проблема
Сейчас “воспроизвести” может использовать устаревший кэш индекса, поэтому треки из уже удалённых из списка папок продолжают попадать в очередь.

### 4.2 Требуемое поведение
- При нажатии кнопки воспроизведения (из главного меню и/или при старте очереди из библиотеки) приложение должно:
  1) Запустить перескан **активных** каталогов из текущего конфига.
  2) Дождаться результата сканирования (асинхронно, без фриза UI).
  3) Построить очередь из **свежего** индекса.
  4) Запустить воспроизведение.
- Если скан уже запущен:
  - не запускать второй,
  - поставить “play pending” и выполнить старт после завершения текущего скана.

## 5) Требование 4: видимый TextInput (текст + курсор)

### 5.1 Требуемое поведение
- В диалоге ввода значений (например “Добавить папку”) должно отображаться:
  - текущий введённый текст,
  - позиция курсора (caret), заметная даже на пустой строке.
- Поддерживаемые действия:
  - ввод символов,
  - backspace/delete,
  - перемещение курсора влево/вправо,
  - submit/cancel как сейчас.

### 5.2 Критерий UX-готовности
- Пользователь видит, что именно он вводит, без необходимости вставки из буфера.

## 6) Совместимость и миграции
- Конфиг должен быть обратно совместим:
  - если в старом `config.yaml` папки были списком строк, они должны импортироваться как папки с `root_only=true`.
- Существующие плейлисты/настройки не должны ломаться.

## 7) Критерии готовности (DoD)
- `.ogg` воспроизводятся на Windows 11 в типичных игровых папках (включая Steam), `.mp3` продолжают работать.
- При удалении папки из списка и нажатии “воспроизвести”, треки из удалённой папки **не** попадают в очередь.
- В списке папок отображается `↓/○`, и переключение флага влияет на глубину сканирования.
- В `TextInput` отображается вводимый текст и курсор.
- Ошибки `ffmpeg`/decode корректно отображаются в `Last error`, приложение не падает.

## 8) Подсказка по текущим местам в коде (для оркестратора)
- Ошибка `.ogg` сейчас формируется при `rodio::Decoder::new(...)` в `app/src/audio/mod.rs` (`decode_file`).
- “Play из меню”: `Action::PlayerLoadFromLibrary { start_index: 0 }` в `app/src/tui/screens/main_menu.rs`.
- Папки в конфиге: `AppConfig.folders` в `app/src/config/mod.rs` + сохранение YAML через `app/src/config/io.rs`.
- Отрисовка списка папок: `app/src/tui/ui.rs` (формат строки `\"{:>2}. {}\"`).
- Обработка `PlayerEvent::Error`: `app/src/tui/app.rs` (кладёт в `state.last_error`).
