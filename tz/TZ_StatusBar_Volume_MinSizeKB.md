# Техническое задание (ТЗ): Volume в статус-строке + min_size в килобайтах

## 0) Цель
Нужно улучшить информативность статус-строки TUI:
- добавить отображение текущей громкости в формате `Volume=NN%`;
- перевести настройку `min_size` на **килобайты** для ввода/хранения/отображения и показывать её в статус-строке как `min_size=1365kb`.

## 1) Термины и единицы
- **kb** в UI/конфиге: *килобайт* = **1024 байта**.
- Внутренние расчёты фильтрации по размеру выполняются в **байтах**.

## 2) Текущее состояние (для привязки к коду)
### 2.1 Где рисуется статус-строка
Статус-строка формируется в [`app/src/tui/ui.rs`](app/src/tui/ui.rs) в функции `draw_status_bar()`.
Сейчас там выводятся поля вида:
`tracks=...  min_size=...  shuffle=...  repeat=...`

### 2.2 Где уже есть управление громкостью
Управление громкостью уже реализовано в плеере через команды:
- `PlayerCommand::SetVolumePercent(u8)`
- `PlayerCommand::AdjustVolumePercent(i8)`

Значение громкости хранится как процент `0..=100`, дефолт задаётся конфигом `audio.default_volume_percent`.

## 3) Требования: отображение громкости (Volume)
### 3.1 Функциональные требования
- **VOL-FR-1**: В статус-строке всегда показывать громкость в формате `Volume={N}%`, где \(N \in [0,100]\).
- **VOL-FR-2**: Примеры допустимых значений (строго): `Volume=75%`, `Volume=0%`, `Volume=100%`.
- **VOL-FR-3**: Значение должно обновляться при изменении громкости хоткеями без перезапуска приложения.

### 3.2 Технические требования (данные)
- **VOL-TR-1**: Пробросить `volume_percent` в снапшот, который получает TUI.
  - Расширить структуру снапшота плеера (например `PlayerSnapshot`) полем `volume_percent: u8`.
  - Заполнять это поле из источника истины (движок плеера).

### 3.3 Технические требования (UI)
- **VOL-TR-2**: Добавить в `draw_status_bar()` вывод поля `Volume={N}%`, используя значение из состояния TUI (`state.player...`).

## 4) Требования: min_size в килобайтах
### 4.1 Пользовательские требования
- **MS-FR-1**: `min_size` должен **задаваться** в TUI в **килобайтах**.
- **MS-FR-2**: `min_size` должен **отображаться**:
  - в статус-строке в формате `min_size={K}kb` (например `min_size=1365kb`);
  - на экране настроек также в килобайтах (и с суффиксом `kb`, либо как число + явная подпись `kb`).
- **MS-FR-3**: Значение по умолчанию — **1024kb**.
- **MS-FR-4**: Значение по умолчанию должно быть **прописано в конфиге** при создании дефолтного `config.yaml` (если там ещё не задано).

### 4.2 Совместимость конфигов (обязательная)
В проекте уже встречается настройка `settings.min_size_bytes`. Требуется совместимость:
- **MS-COMP-1 (read)**: При чтении `config.yaml` поддерживать оба варианта:
  - новый: `settings.min_size_kb`;
  - старый: `settings.min_size_bytes`.
- **MS-COMP-2 (precedence)**: Если в конфиге присутствуют оба поля — приоритет у `settings.min_size_kb`.
- **MS-COMP-3 (write)**: При сохранении конфигурации из приложения записывать **новое поле** `settings.min_size_kb` (и не обязано записывать `settings.min_size_bytes`).
- **MS-COMP-4 (migration)**: Если конфиг старого формата (только `min_size_bytes`), после первого сохранения настроек приложение должно “мигрировать” его на новый формат, добавив `min_size_kb` (с корректным пересчётом) и далее использовать его.

### 4.3 Валидация и пересчёт
- **MS-TR-1**: Внутреннее значение фильтрации должно соответствовать:
  - `min_size_bytes = min_size_kb * 1024`.
- **MS-TR-2**: Валидация:
  - `min_size_kb` — целое неотрицательное значение (рекомендуемый минимум: 0).
- **MS-TR-3**: Отображение:
  - в статус-строке всегда показывать суффикс `kb` в нижнем регистре: `min_size=1024kb`.

## 5) Изменения UI (конкретика)
### 5.1 Строка статуса
Формат статус-строки должен быть расширен, например так:
`... | tracks=24  min_size=1365kb  shuffle=on  repeat=off  Volume=75%`

Требования к форматированию:
- **UI-1**: Сохранить существующую структуру строки и разделитель `|`.
- **UI-2**: Порядок полей допускается менять только минимально; рекомендуется добавлять `Volume=...%` в конец.
- **UI-3**: `min_size` должен отображаться именно как `min_size={K}kb` (не bytes, не `KB`, не `kB`).

### 5.2 Экран настроек
Текущий экран настроек выводит `min_size_bytes`. Требуется:
- показывать `min_size_kb`;
- ввод на редактирование принимать в **kb**;
- при сохранении пересчитывать в байты для фактической фильтрации (см. 4.3), но в конфиг писать `min_size_kb`.

## 6) Изменения конфигурации
### 6.1 Новый ключ
Добавить в `settings` новый ключ:

```yaml
settings:
  min_size_kb: 1024
```

### 6.2 Дефолтный конфиг
- **CFG-1**: При создании дефолтного `config.yaml` значение `settings.min_size_kb` должно быть **явно записано** как `1024`.
- **CFG-2**: Если в проекте существует генерация дефолтов “в памяти”, она должна использовать 1024kb как дефолт.

## 7) Критерии приёмки (Acceptance Criteria)
### 7.1 Volume
- **AC-VOL-1**: При запуске приложения в статус-строке присутствует `Volume=75%` (если `audio.default_volume_percent=75`).
- **AC-VOL-2**: При изменении громкости хоткеями значение в статус-строке изменяется соответственно и ограничивается 0..100.

### 7.2 min_size
- **AC-MS-1**: В статус-строке `min_size` отображается как `min_size=1024kb` (на дефолтном конфиге).
- **AC-MS-2**: Пользователь может изменить `min_size` в настройках, вводя значение в kb; после изменения статус-строка показывает новое значение в `kb`.
- **AC-MS-3 (compat read)**: Старый конфиг с `settings.min_size_bytes` корректно читается и интерпретируется как kb:
  - пример: `min_size_bytes: 1048576` → `min_size_kb: 1024`.
- **AC-MS-4 (migration write)**: После любого сохранения настроек приложение записывает `settings.min_size_kb` в `config.yaml`.

## 8) Затрагиваемые модули (ожидаемо)
- [`app/src/tui/ui.rs`](app/src/tui/ui.rs) — формат статус-строки.
- [`app/src/tui/screens/settings.rs`](app/src/tui/screens/settings.rs) и/или связанный код настроек — отображение/ввод `min_size_kb`.
- [`app/src/config/mod.rs`](app/src/config/mod.rs), [`app/src/config/defaults.rs`](app/src/config/defaults.rs) — новый ключ `min_size_kb`, дефолт 1024kb, совместимость со старым `min_size_bytes`.
- [`app/src/indexer/scan.rs`](app/src/indexer/scan.rs) (или место, где применяется порог) — использовать байтовое значение, вычисленное из kb.
- [`app/src/player/mod.rs`](app/src/player/mod.rs) — проброс `volume_percent` в снапшоты.
