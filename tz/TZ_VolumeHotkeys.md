## ТЗ: регулировка громкости глобальными хоткеями

## 0) Цель
- Добавить в приложение управление **громкостью воспроизведения** через **глобальные хоткеи** (Windows `RegisterHotKey`), чтобы регулировать звук без фокуса на терминале/окне приложения.
- Сохранить подход “всё настраивается через конфигурационный файл” (как и существующие хоткеи).

## 1) Входные данные
- **Платформа**: Windows (глобальные хоткеи через `RegisterHotKey`).
- **Существующая система хоткеев**:
  - обработка: `app/src/hotkeys/mod.rs`, `app/src/hotkeys/logic.rs`
  - схема конфига: `app/src/config/mod.rs` (`HotkeysConfig`, `HotkeysBindings`, `HotkeyModifier`, `HotkeyKey`)
  - дефолты: `app/src/config/defaults.rs`
- **Требования пользователя**:
  - хоткеи по умолчанию: `LeftCtrl + RightShift + PageUp/PageDown`
  - шаг: 5% за нажатие
  - громкость глобальная для всего приложения, хранится только до перезапуска
  - значение по умолчанию: 75%

## 2) Требование A: новые команды “громче/тише”
### 2.1 Ожидаемое поведение
- При нажатии хоткея **громче** громкость увеличивается на **+5%**.
- При нажатии хоткея **тише** громкость уменьшается на **−5%**.
- Громкость ограничена диапазоном **0–100%** (кламп):
  - выше 100% не поднимается,
  - ниже 0% не опускается.
- Громкость **одна на всё приложение** (не на трек).
- Громкость **не сохраняется в конфиг** и после перезапуска возвращается к значению по умолчанию.

### 2.2 Значение по умолчанию
- При старте приложения громкость должна быть установлена в **75%** (если пользователь не менял её в рамках текущей сессии).

## 3) Требование B: глобальные хоткеи (Windows)
### 3.1 Ожидаемое поведение
- Хоткеи должны работать **глобально** (не требуют фокуса на терминале), аналогично текущим глобальным хоткеям приложения.

### 3.2 Дефолтные биндинги
- `LeftCtrl + RightShift + PageUp` → **VolumeUp** (+5%)
- `LeftCtrl + RightShift + PageDown` → **VolumeDown** (−5%)

### 3.3 Важное ограничение платформы (Windows `RegisterHotKey`)
- `RegisterHotKey` оперирует модификаторами уровня **MOD_CONTROL/MOD_SHIFT** и **не позволяет строго различать** левый/правый Ctrl и левый/правый Shift на этапе регистрации хоткея.
- Поэтому в реализации нужно выбрать и зафиксировать один из вариантов:
  - **Вариант 1 (MVP, совместимо с текущим бекендом)**: трактовать требование `LeftCtrl + RightShift` как **`Ctrl + Shift` (любой Ctrl + любой Shift)**, сохранив при этом “человеческие” имена в конфиге для читаемости.
  - **Вариант 2 (строго по ТЗ)**: заменить бекенд глобальных хоткеев на низкоуровневый перехват клавиатуры (например, `WH_KEYBOARD_LL`), чтобы реально различать `LeftCtrl/RightCtrl` и `LeftShift/RightShift` и поддержать именно `LeftCtrl + RightShift`.
 
В данном ТЗ дефолтное сочетание фиксируется как `LeftCtrl + RightShift + PageUp/PageDown`; если выбран Вариант 1 — это должно быть **явно отражено** в релиз-заметке/документации (что фактически работает `Ctrl+Shift+PgUp/PgDn`).

## 4) Требование C: конфигурируемость (как и раньше)
### 4.1 Новые биндинги в конфиге
- В `hotkeys.bindings` должны появиться два новых опциональных поля:
  - `volume_up`: chord (tap-only)
  - `volume_down`: chord (tap-only)
- Если поле отсутствует — используется дефолт из `app/src/config/defaults.rs`.
- Если поле задано `null` — хоткей отключён (аналогично текущей схеме с `Option<...>`).

### 4.2 Пример YAML
```yaml
schema_version: 1
settings:
  supported_extensions: [mp3, ogg]
folders: []
hotkeys:
  bindings:
    volume_up:   { modifiers: [left_ctrl, right_shift], key: page_up }
    volume_down: { modifiers: [left_ctrl, right_shift], key: page_down }
```

### 4.3 Совместимость и устойчивость
- Должна сохраняться совместимость с существующими конфигами:
  - неизвестные поля в `hotkeys.bindings.*` не должны теряться при roundtrip (как сейчас через `#[serde(flatten)]`).
- Если новые поля не распознаны из-за старой версии приложения — конфиг остаётся валидным для старой версии (best-effort, без требований обратной совместимости по коду).

## 5) Технический подход (ориентир)
- Расширить enum `HotkeyKey` для поддержки `page_up/page_down` и обеспечить маппинг в VK (`VK_PRIOR`/`VK_NEXT`) в Windows-слое регистрации.
- Обеспечить различение `LeftCtrl` как модификатора по конфигу (так как требование именно `LeftCtrl`), при этом сохранить поведение остальных хоткеев.
- Добавить новые действия в систему команд/действий приложения:
  - `VolumeUp` / `VolumeDown` (или эквивалентные по стилю существующих `Action::*`).
- Внести runtime-состояние “current_volume_percent” (или эквивалент) со значением по умолчанию 75%, доступное аудио-бэкенду.

## 6) Критерии готовности
- При запуске приложения громкость = **75%**.
- `LeftCtrl+RightShift+PageUp` увеличивает громкость на **5%** за нажатие.
- `LeftCtrl+RightShift+PageDown` уменьшает громкость на **5%** за нажатие.
- Границы 0–100% соблюдаются.
- Хоткеи работают, когда терминал не в фокусе (глобально).
- После перезапуска приложения громкость снова **75%**.
- Хоткеи настраиваются через конфиг (переопределяют дефолт) и могут быть отключены через `null`.

## 7) Мини-тест-план
### 7.1 Дефолт и границы
- Запуск → проверить, что громкость 75%.
- Нажать 1 раз `VolumeUp` → 80%.
- Нажать 1 раз `VolumeDown` → 75%.
- Нажать `VolumeDown` много раз → не ниже 0%.
- Нажать `VolumeUp` много раз → не выше 100%.

### 7.2 Глобальность
- Запустить воспроизведение.
- Свернуть терминал/переключиться в другое приложение.
- Нажимать хоткеи громкости → слышно изменение громкости.

### 7.3 Конфиг
- В конфиге заменить хоткеи на другие (например, `ctrl+right_shift+up/down`) → после перезапуска работают новые.
- Установить `volume_up: null` → хоткей громче не работает, при этом приложение продолжает работать.
